Avvia la dashboard dal terminale con:
uvicorn webapp:app --host 0.0.0.0 --port 8000

Per abilitare la protezione con password imposta la variabile d'ambiente prima del comando:
ETKG_PASSWORD=latuapassword uvicorn webapp:app --host 0.0.0.0 --port 8000

Poi apri il browser su http://<IP-SERVER>:8000. Se hai impostato una password ti verrà mostrato un form di login.
Crea il file /etc/systemd/system/etkg-web.service con questo contenuto:
  • [Unit] → Description=ESET KeyGen Web Dashboard
  • [Service] → WorkingDirectory=/root/etkg
  • ExecStart=/root/etkg/venv/bin/uvicorn webapp:app --host 0.0.0.0 --port 8000
  • Environment="ETKG_PASSWORD=tuapassword"
  • [Install] → WantedBy=multi-user.target
Poi esegui:
systemctl daemon-reload && systemctl enable --now etkg-web

Per verificare lo stato: systemctl status etkg-web
  • ESET HOME Key — genera una license key per ESET Home (antivirus personale). È la modalità più comune.
  • Account ESET HOME — crea un account ESET HOME con trial attivo invece di una sola chiave.
  • Small Business Key — genera una chiave per ESET Small Business Security.
  • Advanced Key — genera una chiave per ESET PROTECT Advanced.
  • ProtectHub Account — crea un account ESET PROTECT Hub. Attenzione: ESET ha una policy aggressiva per gli indirizzi email temporanei in questa modalità.
La maggior parte delle Email API è stata bloccata da ESET. Le opzioni consigliate al momento sono:
  • emailfake — consigliata, funziona nella maggior parte dei casi
  • inboxes — buona alternativa
Se nessuna funziona, abilita l'opzione "Usa email personalizzata (custom email API)" nelle impostazioni avanzate e usa il flag --custom-email-api. Le API mailticking e incognitomail hanno aggiunto captcha Cloudflare e non sono affidabili.
Nelle Impostazioni avanzate trovi la sezione Sorgente Proxy:
  • Inserisci l'URL che restituisce una lista di proxy nel formato ip:porta:utente:password (compatibile con Webshare e altri provider).
  • Clicca Carica proxy: i proxy vengono scaricati e tenuti in memoria.
  • Abilita "Usa pool proxy" per attivare la rotazione automatica.
  • L'URL viene salvato nel config e ricaricato automaticamente al prossimo avvio.
Se non hai un provider, Webshare offre 10 proxy gratuiti al mese sul piano free.
Sì. Quando Ripetizioni è maggiore di 1 e il pool proxy è attivo:
  • Lo script viene eseguito N volte in sequenza, una per ogni ripetizione.
  • Ogni iterazione usa un proxy diverso dal precedente, scelto casualmente dal pool.
  • Per ogni iterazione appare un riquadro Risultato nella pagina (verde = successo, rosso = errore).
  • Questo evita di essere bloccati da ESET per troppe richieste dallo stesso IP.
Se il pool proxy non è attivo, le ripetizioni vengono passate direttamente allo script con --repeat N.
Clicca il pulsante Salva impostazioni nella riga dei controlli principali. Tutte le impostazioni (modalità, browser, email API, ripetizioni, proxy URL, flag avanzati) vengono scritte nel file eset-keygen-config.json nella directory del progetto.

Al prossimo caricamento della pagina i campi vengono pre-compilati automaticamente con i valori salvati.
Il pulsante Stop appare mentre lo script è in esecuzione. Cliccandolo viene inviato un segnale di terminazione al processo e Chrome si chiude. Utile se lo script si blocca, se l'email API non risponde, o se vuoi interrompere un ciclo di ripetizioni prima che finisca.

Dopo lo stop puoi avviare una nuova esecuzione normalmente.
Per impostazione predefinita i file vengono creati nella directory del progetto con nomi nel formato:
ESET KEYS DD.MM.YYYY.txt oppure ESET ACCOUNTS DD.MM.YYYY.txt

La sidebar a destra mostra gli ultimi 5 file generati con i relativi contenuti (email, password, chiave, scadenza). Puoi copiare i singoli valori o l'intero risultato con i pulsanti Copia.

Per cambiare il percorso del file usa il campo File di output nelle impostazioni avanzate.
Su ambienti Linux senza display (server, LXC Proxmox) Chrome richiede alcune configurazioni:
  • Installa Chrome: wget -q -O - https://dl.google.com/linux/linux_signing_key.pub | apt-key add - poi apt install google-chrome-stable
  • Se l'errore è "exit code 127" mancano le dipendenze: apt install -y libnss3 libatk1.0-0 libatk-bridge2.0-0 libcups2 libdrm2 libxkbcommon0 libxcomposite1 libxdamage1 libxrandr2 libgbm1 libasound2
  • In LXC privilegiato aggiungi lxc.cap.drop = (vuoto) per permettere a Chrome di usare namespace.
  • Oppure avvia Chrome con --no-sandbox: il progetto lo include già di default in modalità headless.
  • Imposta il percorso corretto in Percorso browser custom se Chrome non è in /usr/bin/google-chrome.

Questo messaggio indica che il bypass automatico dei cookie non è riuscito, ma lo script continua comunque. ESET in alcuni casi non richiede questo bypass e il processo va a buon fine lo stesso.

Il messaggio è classificato come avviso (WARN) e non come errore (FAILED): non provoca la visualizzazione di un riquadro di errore nei risultati.
  • ACT0 / ACT5: verifica che nessun account ESET HOME sia connesso al prodotto. Se lo è, scollegalo e riprova. Se l'errore persiste, reinstalla ESET e riprova. In alternativa prova la modalità Account invece di Key.
  • ecp4125: errore temporaneo lato ESET. Attendi qualche ora e riprova.
  • In generale: cambia Email API, prova con un proxy diverso, oppure aumenta il numero di ripetizioni per aumentare le probabilità di successo.

Dal terminale, nella directory del progetto:
git pull && systemctl restart etkg-web

Per vedere i log in tempo reale subito dopo il riavvio:
git pull && systemctl restart etkg-web && journalctl -u etkg-web -f

Ctrl+C per uscire dal log.
Dal terminale nella directory del progetto:
git fetch upstream && git rebase upstream/main && git push myfork main

Dove upstream è il remote del progetto originale (shadowcopyrz/etkg) e myfork è il tuo fork. Le modifiche locali (webapp, template) vengono mantenute grazie al rebase.
